=============================================================================
                            HOW IT WORKS
=============================================================================

1. INCIDENT DETECTION
   - User clicks "Simulate Outage" or real incident occurs
   - Incident created with sample logs
   - Stored in Firestore (cloud) or memory (local)

2. AI ANALYSIS
   - Logs sent to AI service
   - Vertex AI Gemini Flash analyzes
   - Root cause identified with confidence score
   - Affected services determined

3. REMEDIATION
   - AI generates step-by-step fix
   - User reviews and approves
   - Remediation executed (simulated or real)
   - Results tracked

4. POSTMORTEM
   - AI generates incident report
   - Timeline documented
   - Lessons learned captured
   - Incident marked resolved

=============================================================================
                          SAMPLE INCIDENTS
=============================================================================

1. DB Connection Timeout
   Logs: Connection pool errors, retries, failures
   Analysis: Pool exhaustion from traffic spike
   Fix: Increase pool size and restart

2. Redis Unavailable
   Logs: Cache misses, fallback to slow path
   Analysis: Redis service down or memory full
   Fix: Restart Redis or clear cache

3. HTTP 503 Spike
   Logs: High queue depth, timeouts, overload
   Analysis: Server capacity exceeded
   Fix: Auto-scale or add rate limiting

4. High Latency
   Logs: Slow queries, SLA breaches, timeouts
   Analysis: Database performance degradation
   Fix: Optimize queries or scale database
